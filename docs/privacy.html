RingBoard Privacy Policy

Effective date: June 9, 2026

RingBoard is a personal hobby project created and operated by Zak Lovell. It is a small desk display, built on an ESP32 microcontroller, that shows the owner's own Oura ring statistics (such as sleep, readiness, activity scores, and heart rate) retrieved from the Oura API. This policy explains what data the project accesses, how that data is handled, and the choices available to its user.

RingBoard is not distributed to anyone else. The developer is the sole user, and the device displays only the developer's own health data.

1. What Data Is Collected

RingBoard retrieves data from the Oura API (version 2) using authorized access to the user's Oura account. Depending on the OAuth scopes granted, this data may include:

  • Daily sleep summaries and sleep scores
  • Daily activity summaries and activity scores
  • Daily readiness summaries and readiness scores
  • Heart rate data
  • Blood oxygen (SpO2) data
  • Stress data

RingBoard does not collect any other personal information. There are no user accounts, no cookies, no analytics, and no advertising of any kind.

2. How the Data Is Used

Data retrieved from the Oura API is used for one purpose only: rendering the user's own health statistics on the device's screen. The data is not used for profiling, marketing, research, or any other purpose.

3. Storage and Retention

Data fetched from the Oura API is held only in the device's volatile memory (RAM) while the device is powered on, and is displayed on its screen. Nothing is written to persistent storage, logged, or retained. When the device loses power or restarts, all fetched data is gone. Fresh data is retrieved from the Oura API as needed while the device runs.

4. Sharing of Data

RingBoard does not share, sell, rent, transmit, or disclose any data to third parties. The only network communication involving health data is the direct, encrypted connection between the device and the Oura API.

5. Third-Party Services

RingBoard relies on the Oura API, operated by Oura Health Oy, as the source of the data it displays. Use of an Oura account and the Oura platform is governed by Oura's own terms and privacy policy, available at https://ouraring.com/privacy-policy. RingBoard does not control how Oura collects or processes data within its own services.

6. Revoking Access

Access to Oura data can be revoked at any time from the connected applications or account settings in the user's Oura cloud account. Revoking access immediately cuts off the device's ability to retrieve any data. Because RingBoard stores nothing persistently, no further deletion steps are required; any data held in memory disappears when the device powers off or restarts.

7. Children's Privacy

RingBoard is a personal device used solely by its adult developer. It is not directed at children and does not knowingly collect any information from children.

8. Changes to This Policy

This policy may be updated from time to time, for example if the project's functionality changes. Any updated version will be posted at the same location with a revised effective date. Continued use of the project after an update constitutes acceptance of the revised policy.

9. Governing Context

RingBoard is a personal, non-commercial project operated in the United States, and this policy is provided in that context.
